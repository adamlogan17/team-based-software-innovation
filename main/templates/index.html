Welcome to this prototype for a system to help children in foster care. Below is a short project description:





This project serves as a prototype for CSC3045, featuring a SQL database and comprehensive CRUD functionality. The system includes a fully operational login mechanism that dynamically adjusts the user interface based on the logged-in user. While the primary purpose of the website's main pages is to showcase the system's UI and operational flow, it's important to note that the text inputs on the journal and social worker chat pages allow users to enter text, which is displayed on the webpage. However, this input doesn't update the database or send messages.

Similarly, the gallery page doesn't directly interact with the database, as would be expected in a more complete solution. Instead, it interfaces with the Dog API to retrieve a list of random dog images for display on the website. It's worth mentioning that occasional "dead links" may result from the API, meaning that some images won't appear. It's crucial to understand that this issue is not a bug in our code but rather a limitation of the API.
